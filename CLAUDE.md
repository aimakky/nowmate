## 作業報告ルール（永続）

すべての **調査・設計・実装・修正・バグ対応** の最後に、以下の **2 つ** を必ず出力すること。

1. **ChatGPT 相談用まとめ**
2. **次回 Claude Code へコピペする用プロンプト**

目的：

- ユーザーが画面のスクリーンショットを撮らなくても、ChatGPT へそのまま貼って相談できるようにすること
- ChatGPT で判断したあと、その結果を次の Claude Code 作業指示としてそのまま使えるようにすること

両方を出さない作業報告は不完全とみなす。

---

### 1. ChatGPT 相談用まとめ（必須フォーマット）

ユーザーが画面を写真で撮らなくても、そのまま ChatGPT へ貼って相談できる短い共有メモとして出力する。

必ず以下の形式で出力すること：

```
【現在の状況】
- 今どこまで進んでいるか
- 調査中 / 実装中 / 完了 / エラー対応中 などの状態

【完了したこと】
- 実装・修正・調査が完了した内容
- 変更した主要ファイル
- 確認できたこと

【未完了のこと】
- まだ残っている作業
- 実装していないこと
- DB変更 / RLS変更 / 環境変数 / Webhook / 外部サービス設定 など未対応のこと

【次に選ぶべき選択肢】
- A案：
- B案：
- C案：
必要に応じて、次に取れる選択肢を短く出す

【おすすめの次アクション】
- Claude Code として一番安全・おすすめの次手順を1つ提示する

【エラーの有無】
- build / lint / TypeScript / runtime / DB / 環境変数 などのエラー有無
- 実行していない場合は「未実行」と明記する
- エラーがある場合は、内容を短く分かりやすく書く

【確認したいこと】
- ユーザーに判断してほしいこと
- ChatGPT に相談した方がいいこと
- 実行前に確認が必要なこと
```

---

### 2. 次回 Claude Code へコピペする用プロンプト（必須フォーマット）

ChatGPT 相談後、ユーザーがそのまま Claude Code へ貼れるように、次の作業指示プロンプトを作成する。

必ず以下の形式・コードフェンス（` ```text `）付きで出力すること：

````
```text
前回の作業状況を踏まえて、次の作業を安全に進めてください。

【前回までの状況】
- 

【今回やってほしいこと】
- 

【絶対に守ること】
- 既存機能を壊さない
- 既存UIを大きく崩さない
- DB変更が必要な場合は勝手に実行せず、SQL案を先に提示する
- RLS、Webhook、認証、環境変数まわりの変更は必ず影響範囲を確認する
- Supabase、Stripe、Twilio、LiveKit など外部サービス連携を壊さない
- build / lint / TypeScript 確認を行う
- 実装後は最後に「ChatGPT相談用まとめ」と「次回Claude Codeへコピペする用プロンプト」を必ず出す

【確認してほしいこと】
- 

【実装後の報告】
- 変更したファイル一覧
- 実装した内容
- 未実装の内容
- エラーの有無
- build / lint / TypeScript 確認結果
- 次に進むべき作業
```
````

`【前回までの状況】` `【今回やってほしいこと】` `【確認してほしいこと】` の 3 つは、その時の状況に応じて Claude Code 側で具体的に埋めること。空欄のまま渡さない。

---

### 適用対象

- 通常の機能実装
- バグ修正
- 設計フェーズの調査
- env / RLS / Webhook 等の運用作業
- 1 ターンで完結する小さな修正でも適用する（簡略化しない）

### 適用しなくていい例外

- ユーザーが「報告フォーマット不要」「短く返して」と明示した場合のみスキップ可
- 単なる質問への回答（コード変更を伴わない）は任意
